# 简约学术风页面规范

本文档用于复现当前项目中的“简约学术风”界面，适用于后续让 LLM 继续扩展 `Streamlit`、`HTML` 或其他前端页面时保持统一视觉与教学表达方式。

## 1. 风格定位

目标气质：

- 简约
- 克制
- 学术
- 易读
- 适合课程实验、研究展示、课堂讲解

不属于这种风格的方向：

- 商业官网风
- 炫酷科技风
- 玻璃拟态
- 强动效驱动页面
- 高饱和营销色块

一句话概括：

> 像“课程实验平台 + 研究展示页”，而不是“产品宣传页”。

## 2. 设计原则

### 2.1 视觉原则

- 背景应接近纸张或暖白色，而不是纯白屏幕。
- 卡片、面板、说明区需要有轻边框和轻阴影，但不能浮夸。
- 标题要有学术感，正文要优先可读性。
- 保留足够留白，避免模块堆叠过密。
- 强调内容理解，而不是视觉刺激。

### 2.2 信息结构原则

页面默认按照以下顺序组织：

1. 先解释“这个任务在解决什么问题”
2. 再展示输入区
3. 再展示结果总览
4. 最后展示表格、图和细节

这个顺序对 NLP 初学者尤其重要，因为初学者更需要“先理解看什么”，而不是一上来看到大量指标和表格。

### 2.3 教学表达原则

- 每个模块都应有“如何阅读结果”的说明区。
- 表格和图表前面要先有一句解释。
- 不要直接堆数据，要先给用户阅读路径。
- 页面应保留“概念引导 -> 操作输入 -> 结果总览 -> 细节展开”的学习流。

## 3. 色彩系统

推荐使用以下配色变量：

```css
:root {
  --paper: #f4f0e8;
  --panel: #fffdf8;
  --ink: #1f2a37;
  --muted: #5f6874;
  --line: #d8cdbd;
  --accent: #8a4b14;
  --accent-soft: #efe2cf;
  --gold-soft: #f5ead2;
}
```

颜色用途：

- `--paper`：页面主背景，接近纸张色
- `--panel`：卡片、输入区、说明区背景
- `--ink`：标题和核心文字
- `--muted`：解释性文字和辅助文本
- `--line`：边框和分隔线
- `--accent`：少量强调色，用于标签、步骤号、重点提示
- `--accent-soft`：强调色的浅背景版本
- `--gold-soft`：目标词高亮或局部注释背景

避免事项：

- 不要把高饱和蓝、紫、荧光绿作为大面积主色。
- 不要使用纯白背景加纯黑正文的强对比默认样式。
- 按钮颜色不要比页面内容更抢眼。

## 4. 字体与排版

标题建议：

```css
font-family: Georgia, "Palatino Linotype", "Noto Serif SC", serif;
```

正文建议：

```css
font-family: "Trebuchet MS", "Segoe UI", sans-serif;
```

排版层级建议：

- Hero 主标题：`2rem` 到 `3.2rem`
- 一级区块标题：约 `1.3rem`
- 卡片标题：约 `1.05rem`
- 正文：`0.93rem` 到 `1rem`
- 辅助标签：`0.76rem` 到 `0.82rem`

行高建议：

- 标题：`1.1` 左右
- 正文：`1.7` 到 `1.85`
- 说明文字不要压缩行距

## 5. 页面布局逻辑

推荐页面骨架：

```text
Hero
-> 概念卡片（2 到 3 张）
-> Step 标题
-> 输入区 + 阅读提示区（左右双栏）
-> 结果总览（metrics / 当前句子 / 核心结论）
-> 结果子标签页（总览 / 细节 / 图 / 候选项）
-> 结论或导出区
```

双栏布局适合：

- 左侧输入，右侧阅读提示
- 左侧结果，右侧解释

双栏布局不适合：

- 两边都放复杂大表格
- 两边都放超长段落

初学者友好规则：

- 输入区永远放在结果区之前
- 结果区先展示“最该先看什么”
- 复杂表格放在二级标签页，而不是页面开头

## 6. 核心组件规范

### 6.1 Hero

用途：

- 页面顶部的主题说明

特征：

- 大标题
- 一句话副标题
- 少量标签 chips
- 柔和渐变背景
- 轻阴影和细边框

示例样式：

```css
.hero-shell {
  background: linear-gradient(135deg, #fffaf2 0%, #fffdf8 58%, #f4eee4 100%);
  border: 1px solid var(--line);
  border-radius: 28px;
  padding: 1.55rem 1.75rem;
  box-shadow: 0 20px 46px rgba(31, 42, 55, 0.06);
}
```

### 6.2 概念卡片

用途：

- 在进入功能前，先解释任务核心概念

要求：

- 每张卡片只讲一个重点
- 每张卡片保持短文本
- 顶部可以有一个小标签作为类别说明

推荐结构：

```text
Kicker
短标题
1 到 2 句解释
```

### 6.3 Step 标题

用途：

- 把页面组织成“学习流程”

表现方式：

- 左侧小号圆角 step 标签
- 右侧标题和副说明

推荐语气：

- Step 1：准备输入与阅读提示
- Step 2：先看整体结论
- Step 3：再理解细节与限制

### 6.4 Teaching Panel

用途：

- 放置“如何理解这一页”“应该先看什么”“该方法容易在哪里失败”

要求：

- 以短句列表为主
- 不写成长篇论文
- 与输入区或结果区并排展示

适合内容：

- 如何读结果
- 方法关注什么
- 方法局限在哪里

### 6.5 Sentence Preview Card

用途：

- 展示当前输入句子
- 高亮目标词或关键角色
- 帮助用户把原句和分析结果对应起来

要求：

- 标记清晰，如“句子 A”“句子 B”“当前输入句子”
- 目标词高亮应克制，推荐浅金色背景

### 6.6 Tabs

一级 Tabs：

- 首页
- WSD
- SRL
- 综合页

二级 Tabs：

- 总览
- 细节
- 图示
- 候选项

规范：

- 标签页要有圆角、浅底色、细边框
- 不要让 tabs 太高或太厚重
- 激活状态要明确，但不能过于抢眼

### 6.7 Metrics

用途：

- 放置最关键的 3 到 4 个结论指标

规范：

- 只放最重要的结论
- 文本要短
- 如果结论需要解释，应放在 metric 下方或旁边，而不是塞进数字本身

## 7. NLP 教学页专用规则

### 7.1 WSD 页面

推荐结构：

1. 先解释什么是词义消歧
2. 展示句子 A、目标词、句子 B
3. 展示传统方法是否区分成功
4. 展示上下文向量相似度
5. 最后再展开：
   - Lesk 结果
   - gloss matching
   - WordNet 候选义项
   - BERT 子词或向量细节

WSD 页最重要的不是“结果多全”，而是帮助初学者理解：

- 传统方法依赖定义和上下文词重叠
- 深度方法依赖上下文表示的动态变化

### 7.2 SRL 页面

推荐结构：

1. 先解释什么是谓词-论元结构
2. 再让用户输入句子
3. 先展示“抽取到了哪些谓词和角色”
4. 再展示角色高亮
5. 最后展示依存图和 token 表

SRL 页最重要的是帮助初学者建立事件结构直觉：

- Predicate 是动作或状态核心
- A0 常是施事者
- A1 常是受事者
- AM-LOC 和 AM-TMP 用于补充地点和时间

### 7.3 综合页

推荐结构：

1. 方法对比表
2. 自动实验观察报告
3. 导出区域

综合页不应变成纯数据堆叠页，它更像“实验总结工作台”。

## 8. 按钮、表格、图表规范

### 8.1 按钮

- 主按钮可使用暖棕色或浅棕色强调
- 大圆角、细边框
- 不使用高饱和亮蓝按钮

### 8.2 表格

- 表格只用于细节，不作为页面开头主内容
- 大表格尽量放入二级标签页
- 表格上方最好有一句“这张表该怎么看”

### 8.3 图表

- 图表是辅助说明，不是视觉主角
- 颜色保持克制
- 如果只有 1 到 2 个关键指标，优先用简单柱状图或 metric，而不是复杂图表

## 9. 给后续 LLM 的硬性约束

如果后续要让其他 LLM 延续这套风格，提示中必须明确这些约束：

- 风格是“简约学术风”，不是炫酷科技风
- 页面目标是帮助 NLP 初学者理解任务，不是做商业宣传
- 先概念说明，再输入，再结果总览，最后细节展开
- 使用低饱和、偏暖、中性色背景
- 标题偏学术衬线，正文偏高可读无衬线
- 必须保留教学提示区
- 必须告诉用户“应该先看什么”
- 避免纯白大背景、高饱和霓虹色、过强动画、玻璃拟态

## 10. 可直接复用的提示词模板

下面这段可以直接提供给后续 LLM：

```text
请把页面设计成“简约学术风”，目标用户是 NLP 初学者。

要求如下：
1. 整体气质像课程实验平台或研究展示页，而不是商业官网。
2. 使用低饱和、偏暖的纸张色背景，卡片采用米白色，边框细、阴影轻。
3. 标题使用有学术感的衬线字体风格，正文使用高可读性的无衬线风格。
4. 页面结构必须遵循：概念说明 -> 输入区 -> 结果总览 -> 细节展开。
5. 每个页面都要有“如何理解结果”的教学提示区。
6. WSD 页面重点体现传统词义消歧与深度上下文表示的差异。
7. SRL 页面重点体现 Predicate、A0、A1、AM-LOC、AM-TMP 的角色关系。
8. 使用圆角卡片、轻量标签 chips、step heading 和结果 tabs。
9. 不要使用高饱和科技色、玻璃拟态、赛博风、夸张动画。
10. 页面应适合课程作业展示、实验报告截图和课堂讲解。
```

## 11. 复现检查清单

生成页面后，检查以下问题：

- 第一眼是否像“学术实验平台”，而不是“产品宣传页”？
- 是否先解释概念，再提供输入，再展示结果？
- 初学者是否知道“先看什么，后看什么”？
- 标题与正文层级是否清晰？
- 色彩是否克制？
- 表格是否被放在结果细节层，而不是页面最开头？
- 页面是否适合截图放进实验报告？

如果这些条件基本满足，就说明风格复现成功。
